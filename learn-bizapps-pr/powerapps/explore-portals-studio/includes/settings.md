## Table permissions

Site administrators can control access to specific pages using content authorization. However, they'll also want to secure custom web applications built with basic forms and lists. To secure these features, table permissions allow granular rights to be granted.

To access table permissions page in Power Apps portal Studio select **Settings** icon on the toolbelt and then select **Table permissions**

![Screenshot of settings panel in Power Apps portals Studio.](../media/settings.png)

This will open a new window where table permissions can be created and managed, and have the web roles assigned to them.

![Screenshot of the table permissions configuration screen in Power Apps portals Studio.](../media/table-permissions.png)

For more information about Table permissions, see [Configure table permissions using portals Studio](/power-apps/maker/portals/configure/entity-permissions-studio).

## Progressive web application

Power Apps portals now support progressive web app (PWA) technology that enables creating an app that will work on all platforms (Android, iOS, Windows, and Chromebooks) and form factors (mobile, desktop, and tablet).

PWAs are built to provide users a native appâ€“like experience. These apps can be installed from the browser or through app stores. This capability enables external and internal customers to use a portal as an app. 

To configure the portal as a PWA select **PWA** icon on the toolbelt then select **Enable PWA** toggle. 

![Screenshot of progressive web app panel in Power Apps portals Studio.](../media/progressive-web-app.png)

On that screen you can configure various PWA properties including its appearance on the mobile devices and offline behavior.  

For more information about building a Power Apps portal as a PWA, see [Portals as progressive web apps (PWAs) overview](/power-apps/maker/portals/progressive-web-apps/?azure-portal=true).
