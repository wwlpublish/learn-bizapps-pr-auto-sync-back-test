In a digitally driven era, the volume of documents, emails, and other types of messages that an organization must process can be overwhelming.

Automating how and where those messages get routed, such as a customer care department, finance group, or marketing team, can help achieve more efficiency.

Using AI Builder entity extraction provides this potential by identifying key elements from the text of those messages or documents and classifying them into predefined categories.

In this unit, you'll learn how the AI Builder entity extraction can process unstructured messages into structured data that can be used to automate routing.

## AI Builder entity extraction

Entity extraction is one of the AI Builder prebuilt models. It's ready to use and doesn't require training.

Before you consider entity extraction, the supported [languages](/ai-builder/prebuilt-entity-extraction?WT.mc_id=DX-MVP-5003600?azure-portal=true#supported-data-format-and-languages) and [entity types](/ai-builder/prebuilt-entity-extraction?WT.mc_id=DX-MVP-5003600?azure-portal=true#supported-entity-types) should be reviewed.
